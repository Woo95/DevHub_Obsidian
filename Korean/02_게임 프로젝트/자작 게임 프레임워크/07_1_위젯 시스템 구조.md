# **위젯 시스템 구조**
---
## **개요**
`CSceneUI`는 **씬 단위로 UI를 관리**하고, **사용자와의 상호작용을 처리**하는 **중심 관리 클래스**입니다. 각 씬에서 UI 구성이 필요할 경우, `CSceneUI`를 상속받아 **씬 전용 UI**를 구성할 수 있습니다. [[07_2_기본 제공 위젯 구성|기본 제공 위젯]]은 `CWidget`을 상속받아 만들어지며, **계층 구조**로 구성됩니다.

**사용자 정의 위젯**은 `CUserWidget`을 상속받아 만들어야 하며, 드래그 등 기본 상호작용 기능은 **이미 구현된 함수**를 통해 **쉽게 적용**할 수 있습니다. **계층 구조**를 활용해 다른 위젯을 조합해 새로운 위젯을 만들 수도 있습니다.

# **위젯 시스템 구조**
---
## **개요**
`CSceneUI`는 **씬 단위로 UI를 관리**하고, **사용자와의 상호작용을 처리**하는 **중심 관리 클래스**입니다. 각 씬에서 UI 구성이 필요할 경우, `CSceneUI`를 상속받아 **씬 전용 UI**를 구성할 수 있습니다. [[07_2_기본 제공 위젯 구성|기본 제공 위젯]]은 `CWidget`을 상속받아 만들어지며, **계층 구조**로 구성됩니다.

**사용자 정의 위젯**은 `CUserWidget`을 상속받아 만들어야 하며, 드래그 등 기본 상호작용 기능은 **이미 구현된 함수**를 통해 **쉽게 적용**할 수 있습니다. **계층 구조**를 활용해 다른 위젯을 조합해 새로운 위젯을 만들 수도 있습니다.

---

## **위젯 시스템 구조도**

![[Widget System Update Flowchart_Kor.png]]

본 구조도는 위젯 시스템의 **업데이트 및 입력 처리 흐름**을 나타냅니다.

- 입력(`CInputManager`) → **호버/클릭 판정**
    
- `CSceneUI` → **호버 및 잡힌 위젯 추적**
    
- `CWidget` → **트리 구조를 통한 자식 위젯까지 업데이트 및 렌더링**
    

---

## **위젯 관련 클래스 목록 및 설명**

### `1. CSceneUI 클래스`

- **역할**
    
    - 씬 단위의 **UI 컨테이너** 역할
        
    - 위젯들의 **생명주기 관리 (Init, Update, Render, Destroy)**
        
    - 마우스 입력을 기반으로 한 **호버, 클릭, 드래그 이벤트 처리**
        
- **핵심 변수들**
    
    - `mWidgets`: 루트 위젯 컨테이너
        
    - `mCurrHovered`: 현재 마우스가 올라간 위젯
        
    - `mHeldWidget`: 마우스로 잡고 있는 위젯
        
- **핵심 함수들**
    
    - `UpdateInput()` : 입력 상태를 바탕으로 호버 및 클릭 이벤트 전달
        
    - `FindHoveredWidget()` : 루트 위젯부터 시작하여 최상위 호버 위젯 탐색
        
    - `BringWidgetToTop(CWidget* widget)` : 특정 위젯을 컨테이너 최상단으로 이동 (Z-Order 효과)
        

---

### `2. CWidget 클래스`

- **역할**
    
    - 모든 UI 요소의 **기본 클래스**
        
    - **트리 구조 기반**으로 부모-자식 관계 형성
        
    - 공통적인 업데이트/렌더링 및 상호작용 인터페이스 제공
        
- **핵심 변수들**
    
    - `mSceneUI` : 자신이 속한 `CSceneUI`
        
    - `mParent`, `mChilds` : 부모 및 자식 위젯 포인터
        
    - `mIsInteractable` : 상호작용 가능 여부
        
    - `mWidgetHovered`, `mWidgetHeld` : 마우스 상태 플래그
        
- **핵심 함수들**
    
    - `Update(deltaTime)` : 자기 자신 및 자식 위젯 업데이트
        
    - `Render(SDL_Renderer* renderer)` : 자기 자신 및 자식 위젯 렌더링
        
    - `HandleHovered() / HandleUnhovered()` : 호버 상태 변화 시 호출 (파생 클래스에서 오버라이드)
        
    - `AddChild(CWidget* child)` : 트리에 자식 추가 및 Transform 계층 연결
        

---

### `3. CUserWidget 클래스`

- **역할**
    
    - 사용자 정의 UI 위젯의 기반 클래스
        
    - 드래그 및 이동과 같은 추가적인 UI 인터랙션 제공
        
- **핵심 변수들**
    
    - `mIsMovable` : 드래그 가능 여부
        
    - `mDragOffset` : 드래그 시 마우스 오프셋 값
        
- **핵심 함수들**
    
    - `HandleDragging(mousePos, isPressed, isHeld, isReleased)` :
        
        - 눌렀을 때 : 드래그 시작 및 `BringWidgetToTop()` 호출
            
        - 이동 중 : 마우스 위치에 맞춰 위젯 이동
            
        - 놓았을 때 : 드래그 종료
            

---

## **위젯 입력 처리 흐름**

1. `CInputManager` : 마우스 좌표 및 버튼 상태 제공
    
2. `CSceneUI::UpdateInput()` :
    
    - 현재 호버된 위젯 탐색 (`FindHoveredWidget`)
        
    - 이전 호버 위젯과 비교 후 `HandleHovered` / `HandleUnhovered` 호출
        
3. `CWidget` : 상호작용 가능 여부(`mIsInteractable`)에 따라 이벤트 수신
    
4. 필요 시 `CUserWidget` : 드래그 이동 처리
    

---

## **맺는 말**

> 위젯 시스템은 **트리 구조 기반의 계층형 UI 관리**를 제공하며,  
> `CSceneUI`를 통한 **중앙 집중식 입력 처리**와  
> `CWidget`의 **재귀적 업데이트 및 렌더링 구조**로  
> **확장성과 유지보수성**이 높은 UI 아키텍처를 구성합니다.  
> 또한 `CUserWidget`을 통해 **드래그 및 이동과 같은 사용자 정의 상호작용**을 유연하게 구현할 수 있습니다.
